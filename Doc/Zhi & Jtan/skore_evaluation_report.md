## 6. Methodology Notes

This evaluation follows the **Probabl Skills Pack v0.6.0** methodology:

- **`explore-ml-data`** — Full EDA on catalogue metadata: null rates, format distribution, category balance, publisher concentration
- **`build-ml-pipeline`** — TF-IDF vectorisation (8,000 features, unigrams + bigrams, sublinear TF scaling) combined with OneVsRest Logistic Regression for multi-label classification
- **`evaluate-ml-pipeline`** — 5-fold stratified cross-validation with micro, macro, weighted and per-category metrics; Hamming loss reported for multi-label quality
- **`audit-ml-pipeline`** — Low-confidence predictions flagged; per-category support checked for class imbalance; governance findings surfaced as policy recommendations
- **`iterate-ml-experiment`** — Decision threshold tuned per category to optimise F1 on imbalanced classes (e.g. Environment, Poverty)

**Reproducibility:** All random seeds fixed (`random_state=42`). Pipeline is fully open source and can be re-run against any future version of the DFL catalogue.
